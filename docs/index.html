Calculator is an app developed for IoT engineers such as programmer and systems engineer.
All of data handled by IoT is represented in binary format (0 and 1).
This app calculates typical data such as "Numeric", "UNIX time", "IP Address", and "Characters" and displays the data as bit array or hexadecimal dumps.

Numeric
Selects the type of numerical value from the following types:
  - Decimal / Double floating point (Double)
  - Decimal / 64 bit Signed integer (Int64)
  - Hexadecimal / Un-signed integer (UInt64)
Double can handle up to 15 digits (including up to 10 decimal places).
Int64/UInt64 can handle numerical values within the range of 64 bits, and in addition to the four basic arithmetic operations, modulo operations and various bit operations are possible.

UNIX Time
Converts the number of seconds elapsed since UNIX epoch, and gregorian date time to each other.
The date time are displayed in either local time or Coordinated Universal Time (UTC).
You can enter date time with the current time, the boundaries of the year 2038/2106 problems, and the calendar.

IP Address
Calculates the network address, subnet mask, and broadcast address from the IPv4 CIDR format.
The IPv6 address format will be supported in the next version.

Characters
Converts the entered character string to binary data (up to 64 KB), and displays in hexadecimal dump.
In addition, a binary data can be edited with an binary editor and can be converted back to a character string.
The encoding of multibyte code in character string data is UTF-8 (other character encodings are not supported).


Specification / notes for numeric:

  • Priority of all operators are as same, and always evaluate from the beginning (ex. 3 + 2 * 2 = 10).
  • The overflow operation (is a operation that truncates the overflow / underflow part) is not performed, and an operation that results in overflow is canceled.
  • Division and modulo operation by 0 are canceled.
  • In Double type, doesn't use exponential notation (eg. 1.234e+16). Therefore, the calculation is performed within the range of 15 digits, which is the number of effective digits of double precision.
  • The effective digits in the decimal part of double type is the 10th place.
  • In Double type, an operation that results greater than 999999999999999 (or less than -999999999999999) are canceled.
  • In Double type, an operation that results in a positive number less than 0.0000000001 (or a negative number greater than -0.0000000001) do not result in an error, and the result is zero.
  • Bit array of Double type is displayed in double-precision IEEE 754 format.
  • If you switch the type of numeric value, the memory will be cleared.
  • When switching from Double to Int64 or UInt64, the decimal part is truncated.
  • When switching from Double or Int64 to UInt64, a negative value will be 0.
  • When switching from Int64 or UInt64 to Double, a value greater than 9999999999999999 (or less than -999999999999999) will be 0.0.
  • When switching from UInt64 to Int64, the value above the maximum value of Int64 (2 to the 63rd power -1) will be 0.
